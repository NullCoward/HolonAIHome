Glossary

Key terms and concepts in the HolonicEngine.

A

Action

A callable function attached to a Holon. Actions are how the AI affects the world — callback functions that execute when the AI requests them.

Action History

A log of all actions executed, including triggers and LLM-directed actions. The agent reviews this to understand what happened.

Agency

The capacity to act independently and make choices. Holonic Agents have true agency — they make decisions about their own behavior, resources, and development.

C

Compiled Cognition

The process of converting conscious reasoning into automated scripts. Like how humans develop habits, agents compile repeated patterns into triggers.

Context Window

The maximum tokens an LLM can process in one call. The HolonicEngine solves context limits through the Memory Lattice.

D

Demand Sphere

Work that requires active LLM reasoning (token-expensive). Contrast with automated sphere.

Discretionary Time

Token budget remaining after work is done, available for self-directed activity like research, learning, or memory organization.

E

Expert Holon

A specialized holon that has taught itself about a specific domain (like an API) and serves as the agent's interface to that domain.

H

Heartbeat

The cognitive clock — a regular tick that drives the agent's life rhythm. Each heartbeat, the agent checks triggers, handles work, and potentially uses discretionary time.

Holon

The fundamental building block. A portable AI context capsule combining Purpose (who am I), Self (what do I know), and Actions (what can I do).

Holonic Agent

An AI agent built on the HolonicEngine architecture. Distinguished by true agency — persistent memory, self-awareness, autonomous behavior.

HUD

Heads-Up Display. The resource monitor that shows the agent its token usage, budget, and optimization options.

I

Interactive Summary

A summary that serves as a door to more detail, not a replacement for it. Can be expanded when needed.

Internal Dialogue

Querying holons within the lattice, like consulting parts of your own mind.

M

Memory Lattice

The hierarchical structure of holons that stores all the agent's knowledge. Navigable, expandable, persistent.

Model-Agnostic

The architecture works with any LLM (Claude, GPT, etc.) because state lives outside the model.

P

Purpose

The first component of a Holon. Defines who the holon is, its goals, constraints, and behavioral guidance.

Portable Cognition

The ability to move an agent's entire "mind" between systems, models, or storage backends.

S

Self (State)

The second component of a Holon. Contains current context, memory, child holons, and dynamic data.

Spawn

Creating a new holon, often as a child of an existing one.

T

Token Budget

The maximum tokens allocated for a specific purpose (per heartbeat, per holon, per task).

Trigger Script

A pre-written automation that executes without LLM cost when its trigger condition is met.

Taglines

"Agents with Agency"

Holonic Agents actually have autonomy — they make decisions, manage resources, and develop over time.

"AI wearing code like a body"

The LLM is a reasoning engine; the Holon is the body it inhabits.
